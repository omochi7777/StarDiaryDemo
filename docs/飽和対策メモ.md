# ⭐ 星の飽和対策メモ

> 作成日: 2026-02-26
> ステータス: 検討中

## 背景

星が増えていくと空が飽和状態になる。長期的に使い続けられる仕組みが必要。

---

## 採用候補

### 🏆 案E: 星座完成で「合体」表示（優先度: 高）

完成した星座を1つのシンボル（星座マーク）に圧縮して表示する。

#### 基本アイデア
- 完成した星座の星5つ → 1つのシンボルアイコンに変わる
- シンボルは星座の中心位置に表示
- タップするとポップアップで元の星の詳細（達成内容リスト）が見られる
- 空間が空いて新しい星を配置できるようになる

#### 検討ポイント
- [ ] シンボルのデザイン（実在星座 / 架空星座で分ける？）
- [ ] 合体アニメーション（星が中心に集まって光る → シンボルに変化？）
- [ ] シンボルのサイズ（固定 or 含まれる星の数で変わる？）
- [ ] 合体のタイミング（完成直後？ 一定時間後？ 手動切り替え？）
- [ ] シンボルをタップしたときのUI（ポップアップ？ モーダル？ 図鑑へジャンプ？）
- [ ] 合体した星座の線はどうする？（非表示にする）

#### 実装メモ
- DB側: `constellations` テーブルに `collapsed: boolean` フラグを追加
- 描画側: `collapsed` な星座は星・線を描かず、代わりにシンボルを描画
- starEngine: 合体後は星座の中心座標を計算して保持

### 🌸 季節で空の色が変わる（優先度: 中）

現実の季節や使用期間に応じて空のグラデーションを変化させる。

#### 基本アイデア
- 空のグラデーション（`drawSkyGradient`）を季節ごとに変える
- 春: やや明るい紺〜藤色、夏: 深い紺〜群青、秋: 暖色混じりの紺、冬: 冷たい深青
- 雲の色味や透明度も季節で変えると雰囲気が出る
- 地面の色（丘の色）も連動して変えられる
- 空色変更は「プリセット選択のみ」にする（自由色指定は当面入れない）

#### 検討ポイント
- [ ] 季節の判定方法（現実の日付？ アプリ内の進行度？）
- [ ] 季節の切り替えタイミング（徐々にフェード？ 一瞬で切り替え？）
- [ ] 季節ごとの配色テーブルを定義する
- [ ] 季節限定の演出（春: 桜の花びら、夏: 蛍、秋: 流れ星多め、冬: 雪？）
- [ ] ユーザー向けプリセット切り替えUI（おまかせ / 春 / 夏 / 秋 / 冬）を用意する

#### 実装メモ
- `renderer.ts` の `drawSkyGradient` に季節パラメータを渡す
- 設定値 `skyThemePreset` を保持（`auto | spring | summer | autumn | winter`）
- 配色テーブル例:
  ```
  spring: ['#0c1035', '#1a1d50', '#2d2858', '#3a2d55']
  summer: ['#060b1e', '#0d1539', '#162050', '#1a2555']  // 現在の配色
  autumn: ['#0f0a1a', '#1e1530', '#2a1d3a', '#352838']
  winter: ['#050a20', '#0a1540', '#152558', '#1a2d60']
  ```

### ⚙️ 描画・探索の最適化（優先度: 中）

星数が増えたときの処理負荷を下げ、操作感を維持する。

#### 基本アイデア
- `loadData` で線を復元するとき、`stars.find(...)` 連打をやめて `Map<starId, Star>` で O(1) 参照にする
- ホバー判定（最近傍探索）に空間分割（グリッド/ハッシュ）を導入し、全星走査を避ける
- 必要であれば星座単位の描画対象キャッシュを持ち、毎フレーム計算を減らす

#### 検討ポイント
- [ ] `Map` の構築タイミング（`loadData` 時のみ / 差分更新）
- [ ] 空間分割の粒度（固定セルサイズ / ズーム連動）
- [ ] 実装複雑度に対して体感改善が十分か（星数100, 300, 500で計測）

#### 実装メモ
- `App.tsx`:
  - `loadData` 内で `const starById = new Map(stars.map((s) => [s.id!, s]))` を作って線復元に使う
  - ホバー判定用インデックス（例: `Map<string, Star[]>`）を保持し、近傍セルのみ検索する
- `renderer.ts`:
  - 描画順は現状維持しつつ、将来的に星座ごとのバッチ描画を検討
