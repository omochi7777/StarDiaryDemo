# Star Diary アプリ構成再現ガイド

このドキュメントは、現状の `stardiary` アプリを同等構成で再現するための技術メモです。

## 1. 技術スタック

- フロントエンド: React 19 + TypeScript
- ビルド/開発: Vite 6
- 永続化: IndexedDB（Dexie 4）
- PWA: `vite-plugin-pwa`
- 描画: Canvas 2D（独自レンダラー）
- フォント: Google Fonts `Zen Maru Gothic`

`package.json` の主要依存:

- dependencies
  - `react@^19.0.0`
  - `react-dom@^19.0.0`
  - `dexie@^4.0.11`
- devDependencies
  - `vite@^6.1.0`
  - `typescript@~5.7.2`
  - `@vitejs/plugin-react@^4.3.4`
  - `vite-plugin-pwa@^0.21.1`

## 4. 実行時アーキテクチャ

### 4.1 起動フロー

1. `src/main.tsx` で `App` をマウント
2. `App` 初期化時に:
   - Canvasサイズ/DPR設定
   - `initRenderer()` 実行
   - IndexedDB から星・線を `loadData()` で読み込み
   - `requestAnimationFrame` で描画ループ開始（`viewMode === 'sky'` の間）
3. `localStorage` の `stardiary.skyThemePreset` を読み、空テーマを復元

### 4.2 画面モード

- `sky`:
  - Canvasの星空表示
  - 入力トレイ（できたこと入力）
  - 空色プリセット選択
  - UI非表示時のシェアボタン
- `zukan`:
  - 完成した星座一覧カード
  - 星数表示
  - 全データクリア

### 4.3 状態管理（`App.tsx`）

- React state: 入力文字列、送信中、画面モード、図鑑データ、星数、UI表示状態など
- `renderStateRef`: 描画に必要な実体データ（星・線・カメラ・エフェクト）を保持
- DB更新後に `loadData()` を再実行し、描画状態を同期

## 5. 永続化モデル（IndexedDB / Dexie）

DB名: `StarDiaryDB`  
定義ファイル: `src/db.ts`

テーブル:

- `achievements`
  - `id` (PK)
  - `text`
  - `createdAt`
  - `starId`（後から紐付け）
- `stars`
  - `id` (PK)
  - `x`, `y`（Canvas座標）
  - `gridX`, `gridY`（論理グリッド）
  - `brightness`, `size`, `color`
  - `achievementId`
  - `constellationId`
  - `createdAt`
- `constellationLines`
  - `id` (PK)
  - `constellationId`
  - `fromStarId`, `toStarId`
- `constellations`
  - `id` (PK)
  - `name`
  - `isReal`
  - `starCount`
  - `completedAt`

## 6. 星追加ロジック（`src/starEngine.ts`）

`addStar(achievementId, canvasWidth, canvasHeight)` の処理:

1. 空きグリッドを検索（12x8）
2. 未完成星座を取得（なければ新規作成）
3. 新規星座作成時のルール:
   - 完成済み星座数ベースで、5星座ごとに `isReal = true`
   - 名前は `constellationNames.ts` から抽選
4. 配置セル選定:
   - 星座の1個目は他星座と距離を取りつつアンカー近傍
   - 2個目以降は同一星座の重心周辺を優先
5. `gridToCanvas()` で座標化し、色/明るさ/サイズをランダム付与して星を保存
6. 同一星座内で最近傍の星へ線を接続（交差回避を優先）
7. `starCount` 更新、5個到達で `completedAt` を設定
8. `achievement.starId` を更新して紐付け完了

補足:

- 1星座あたりの完成条件は `STARS_PER_CONSTELLATION = 5`
- 交差判定は線分交差の簡易実装

## 7. 描画パイプライン（`src/renderer.ts`）

1フレームごとに以下順で描画:

1. 背景グラデーション（季節テーマ）
2. 雲
3. 背景星
4. カメラ適用ワールド空間へ変換
5. 星座線
6. 星（発光・瞬き）
7. 紙片エフェクト（入力→星化）
8. 星座完成エフェクト（線描画→フラッシュ→名称表示）
9. カメラ解除
10. 地面と木シルエット
11. ホバー時ツールチップ

## 8. ユーザー操作

- マウス/タッチ:
  - ドラッグでパン
  - ホイール/ピンチでズーム（範囲 0.35〜2.5）
  - ダブルクリック/ダブルタップでカメラリセット
- 星空タップ:
  - UI表示/非表示を切替
- UI非表示時:
  - 星空スクリーンショットを Web Share API かダウンロードで保存

## 9. PWA構成（`vite.config.ts`）

- `VitePWA` で `registerType: 'autoUpdate'`
- マニフェスト:
  - name: `Star Diary - できたことが星になる`
  - display: `standalone`
  - orientation: `portrait`
  - theme/background: `#0a0e27`
- アイコン:
  - `public/pwa-192x192.png`
  - `public/pwa-512x512.png` (`maskable` 対応)
- Workbox runtime cache:
  - `https://fonts.googleapis.com/*` を `CacheFirst`

## 10. 再現チェックリスト

- `npm run dev` で初期表示できる
- 入力送信で星が増える
- 5件で星座完成演出が出る
- 図鑑タブで完成星座が一覧化される
- リロード後もDBデータが保持される
- 空色プリセットが `localStorage` で復元される
- UI非表示時に星空シェア/保存が動く
